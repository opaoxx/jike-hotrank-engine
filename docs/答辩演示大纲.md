# Day6 答辩 PPT 大纲

建议 10 页以内，控制在 8 到 10 分钟讲完，把剩余时间留给 Demo 和问答。

## 1. 项目背景

- 兴趣社区热点榜单需要分钟级反映互动变化。
- 榜单不仅看互动数量，还要考虑时效性、真实性和圈子差异。
- 本项目用 Spring Boot、MyBatis、MySQL 实现实时热点榜单引擎。

## 2. 总体架构

建议放 `docs/系统架构说明.md` 中的数据流图：

```text
互动事件 -> 防刷校验 -> 事件流水 -> 定时聚合 -> topic 当前热度 -> 榜单缓存 -> 查询接口/SSE
```

讲解重点：

- 写路径保留完整行为审计。
- 读路径通过榜单缓存降低数据库压力。
- 定时任务使用 MySQL named lock 避免多实例重复执行。

## 3. 数据模型

展示核心表：

- `topic`：当前热度和状态。
- `interaction_event`：有效互动流水和防刷权重倍率。
- `user_behavior`：有效/无效行为审计。
- `topic_score_snapshot`：历史榜单快照。
- `user_circle_preference`：个性化圈子偏好。

## 4. 热度算法

```text
score = weightedInteractionScore / (publishHours + 2)^1.8
```

讲解重点：

- 评论、转发比点赞权重更高。
- 发布时间越久，分母越大，热度自然衰减。
- 防刷风险通过 `weight_multiplier` 降低贡献。

## 5. 多维榜单

展示接口矩阵：

- 全站热榜：平台整体热点。
- 圈子热榜：兴趣圈内独立排序。
- 新星榜：24 小时内新内容。
- 飙升榜：最近 1 小时增速。
- 个性化榜单：按用户圈子偏好重排。

## 6. 防刷与缓存

防刷：

- 同用户同话题频率限制。
- 设备指纹多账号风险降权。
- 话题异常突增标记。

缓存：

- 本地缓存降低榜单读压力。
- 空值缓存避免穿透。
- 随机 TTL 降低雪崩风险。
- 聚合、屏蔽、恢复时主动失效。
