## Redis 对比榜单观察

Redis 全站榜同步后 Top3 与 MySQL 全站榜一致：

| 排名 | 话题 | 分数 |
| ---: | --- | ---: |
| 1 | 小米SU7实车体验 | 0.0004 |
| 2 | 华为Mate70曝光 | 0.0003 |
| 3 | 上海米其林餐厅打卡 | 0.0003 |

同时观察到：当多个话题分数接近或相同，MySQL 的 `ORDER BY current_score DESC` 和 Redis ZSet 的同分成员排序可能出现局部顺序差异。后续如果要让两条排名通道完全一致，可以加入二级排序规则，例如 `current_score DESC, interaction_count DESC, id DESC`，并在 Redis member score 或查询后重排中体现相同规则。

## 额外发现

- `topicId=1` 在模拟前已经是 `status=2` 待审核状态，因此不会出现在全站榜和圈子榜。
- 第一次脚本尝试使用 `curl -d` 在 PowerShell 中发送 JSON，因转义问题导致后端收到非法 JSON；改用 `Invoke-RestMethod -ContentType 'application/json; charset=utf-8'` 后全部写入成功。
- 当前全站榜分数保留 4 位小数，多个话题显示为 `0.0002` 时，实际数据库值可能仍有更细粒度差异。答辩时建议同时说明互动数和飙升榜，避免只看展示分数误判排序。

## 可复现操作

1. 记录当前榜单：

```powershell
curl.exe "http://localhost:18080/api/ranking/global?limit=10"
curl.exe "http://localhost:18080/api/ranking/circle/5?limit=5"
```

2. 写入不同类型互动：

```powershell
$body = @{
  topicId = 13
  userId = 910001
  interactionType = 5
  deviceFingerprint = "sim_probe_001"
  ipAddress = "10.88.13.10"
} | ConvertTo-Json -Compress

Invoke-RestMethod `
  -Uri "http://localhost:18080/api/interaction" `
  -Method Post `
  -ContentType "application/json; charset=utf-8" `
  -Body $body
```

3. 触发聚合和 Redis 同步：

```powershell
curl.exe -X POST "http://localhost:18080/api/ops/heat-aggregation?token=ops_demo_token"
curl.exe -X POST "http://localhost:18080/api/redis-ranking/sync?token=ops_demo_token"
```

4. 观察榜单变化：

```powershell
curl.exe "http://localhost:18080/api/ranking/global?limit=10"
curl.exe "http://localhost:18080/api/ranking/surging?limit=8"
curl.exe "http://localhost:18080/api/redis-ranking/global?limit=10"
```
